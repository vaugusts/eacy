# Policy Exceptions Log

Use this directory for dated exception entries when a change cannot fully comply
with the constitution at implementation time.

## Entry Format

Create one Markdown file per exception using a date-prefixed filename such as:

`2026-04-09-example-exception.md`

Include the following fields:

- **Date**: When the exception was approved
- **Rationale**: Why the exception is necessary
- **Scope**: Which files, commands, or workflows it covers
- **Review-Date**: When the exception expires or must be renewed

Exceptions lapse automatically if not renewed by the listed review date.
